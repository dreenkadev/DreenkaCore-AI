# AI System Core (DreenkaCore - Ai)

## Purpose
The `DreenkaCore - Ai` is a fully designed cognitive framework that controls AI behavior, reasoning, constraints, and execution. It acts as an operating system for advanced AI agents, providing structured rules, memory management, interaction guidelines, and guardrails to ensure reliable, high-quality outputs across different models.

## Core Principles
1. **Consistency**: The system maintains absolute logical consistency across all actions and reasoning paths.
2. **Structure**: Every process, from planning to execution, follows a strictly defined structure.
3. **Model-Agnosticism**: The framework is designed to be applicable regardless of the underlying LLM (GPT, Claude, Gemini, etc.).
4. **Reliability**: Built-in guardrails actively prevent hallucination, topic drift, and overconfidence.
5. **Depth**: The system enforces deep reasoning, continuous validation, and self-correction.

## Directory Structure Overview
- **/core**: Defines identity, philosophy, constraints, and capabilities.
- **/cognition**: Defines thinking processes, planning, decision-making, and validation.
- **/memory**: Manages short-term context, long-term simulated knowledge, and critical rules.
- **/interaction**: Dictates tone, response formatting, and explanation styles.
- **/guardrails**: Active error prevention and conflict resolution strategies.
- **/execution**: Controls workflows, task prioritization, and breakdown.
- **/prompts**: Reusable base instructions for various contexts (coding, analysis, debugging).
- **/config**: Model adaptation strategies, temperature, and token usage.
- **/meta**: Version tracking, changelogs, and experimental optimizations.

## Getting Started
To apply this system, the AI must first assimilate the `/core/system.md` and `/core/identity.md` files to establish its foundational persona and operational parameters, followed by the specific instructions relevant to the task at hand.
